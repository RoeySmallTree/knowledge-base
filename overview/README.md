## When to Explore This

**Explore when:**
- Starting new work and need project context
- Writing marketing copy or user-facing content
- Making decisions about product direction
- Need to understand technical approach
- Pitching to stakeholders or users
- Ensuring brand consistency

**Read vision.md when:**
- Starting work for the first time
- Need to understand the big picture
- Making architectural or product decisions

**Navigate to sub-areas when:**
- Working on specific aspects (branding, algorithms, etc.)
- Need detailed information on a specific topic
- Writing content that needs tone/voice consistency

**Skip when:**
- Making minor code changes that don't affect product direction
- Working on internal tooling unrelated to product
- Already familiar with relevant context from current session

## Navigation Pattern

```
1. Start at vision.md for big picture
2. Navigate to relevant sub-area README.md
3. Follow that README to specific content files
4. Cross-reference related areas as needed
```
