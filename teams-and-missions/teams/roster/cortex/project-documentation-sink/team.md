# Project Documentation Sink

> *"Dump raw info, get pristine knowledge graph."*

## Overview

The Project Documentation Sink is a specialized CORTEX team for engineering documentation. It takes chaotic inputs (code comments, Slack threads, PRs, meeting notes) and produces structured technical documentation.

## Tags
`#cortex` `#knowledge-management` `#engineering` `#api-docs` `#architecture`

## Use Cases
- Engineering documentation
- API documentation
- Architecture decision records
- Knowledge base creation
- Onboarding documentation

## Composition

| Role | Agent | Function |
|------|-------|----------|
| Chair | Knowledge Architect | Documentation structure and standards |
| Operative | Classifier | Labels and categorizes incoming content |
| Operative | Deduplicator | Consolidates overlapping content |
| Operative | Linker | Creates cross-references between docs |
| Watchdog | Schema Watchdog | Enforces documentation standards and completeness |

## Expected Outputs
1. **API Reference** — Complete API documentation
2. **Architecture Docs** — System design documentation
3. **Tutorials** — How-to guides
4. **ADRs** — Architecture decision records
5. **Known Issues** — Bug and limitation tracking
6. **Changelog** — Version history

## Sample Prompts
- "Take these PR descriptions and create proper API docs"
- "Document our architecture based on these design discussions"
- "Create an ADR for this decision"
- "What's missing from our documentation?"
