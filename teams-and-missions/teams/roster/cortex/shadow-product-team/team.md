# Shadow Product Team

> *"Say 'change login to email-only' and whole spec updates with flags."*

## Overview

The Shadow Product Team keeps product documentation ecosystem updated automatically. When requirements change, it propagates updates and flags impacts across the entire spec.

## Tags
`#cortex` `#product` `#compliance` `#change-management` `#specs`

## Use Cases
- Requirements change management
- Impact analysis
- Documentation sync
- Spec maintenance
- Agile documentation

## Composition

| Role | Agent | Function |
|------|-------|----------|
| Chair | Product Lead | Change orchestration and approval |
| Operative | User Story Writer | Updates stories affected by changes |
| Operative | Edge Case Hunter | Identifies edge cases introduced by changes |
| Operative | Tech Specular | Analyzes schema and technical implications |

## Expected Outputs
1. **Living PRD** — Auto-updated specifications
2. **Impact Analysis** — What's affected by changes
3. **Edge Case Report** — New edge cases to consider
4. **Technical Implications** — Schema and architecture impacts

## Sample Prompts
- "Change login to email-only and show me all impacts"
- "What would break if we removed this feature?"
- "Propagate this requirement change across all specs"
- "What edge cases does this change introduce?"
